# Multi-Account WhatsApp API Server — Full System Prompt

> **Stack**: Node.js · TypeScript · Baileys (@whiskeysockets/baileys) · Express · PostgreSQL (Knex) · Filesystem session storage

---

## Project Overview

Build a production-ready, multi-tenant WhatsApp API server that:

- Allows **multiple users** to register and manage API keys/tokens
- Allows **each user** to connect **multiple WhatsApp numbers** simultaneously
- Exposes all WhatsApp functionality over a clean **REST + Webhook API**
- Uses **Baileys** (`@whiskeysockets/baileys`) as the underlying WA socket library
- Persists session credentials so numbers don't need to be re-scanned after restart
- Handles reconnections, QR refresh, and socket lifecycle automatically

## Database Schema (PostgreSQL)

Use Knex migrations. All tables use `uuid` primary keys generated by `gen_random_uuid()` (requires `pgcrypto` extension, or use `uuid-ossp`). The `instances.status` column uses a native Postgres `ENUM` type. `webhook_events` uses `JSONB` for efficient querying.

Enable the required extension in your first migration:
```sql
CREATE EXTENSION IF NOT EXISTS "pgcrypto";
```

### `users`
| Column | Type | Notes |
|---|---|---|
| id | UUID PK DEFAULT gen_random_uuid() | |
| email | VARCHAR(255) UNIQUE NOT NULL | |
| password_hash | VARCHAR(255) NOT NULL | bcrypt |
| created_at | TIMESTAMPTZ DEFAULT NOW() | |

### `api_keys`
| Column | Type | Notes |
|---|---|---|
| id | UUID PK DEFAULT gen_random_uuid() | |
| user_id | UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE | |
| key_hash | VARCHAR(64) NOT NULL | SHA-256 hex, 64 chars |
| label | VARCHAR(100) NOT NULL | e.g. "production", "dev" |
| last_used_at | TIMESTAMPTZ NULL | |

Index: `CREATE INDEX ON api_keys(key_hash) WHERE revoked = FALSE;`

### `instances`
| Column | Type | Notes |
|---|---|---|
| id | VARCHAR(150) PK | `{userId}_{slug}`, e.g. `usr_abc_main` |
| user_id | UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE | |
| slug | VARCHAR(50) NOT NULL | user-defined name, unique per user |
| phone_number | VARCHAR(30) NULL | filled after first connect |
| status | instance_status NOT NULL DEFAULT 'disconnected' | Postgres ENUM |
| webhook_url | VARCHAR(500) NULL | per-instance webhook endpoint |
| webhook_events | JSONB NOT NULL DEFAULT '[]' | subscribed event types |
| webhook_secret | VARCHAR(64) NULL | HMAC signing secret |
| sessions_dir | VARCHAR(500) NOT NULL | absolute path on disk |
| created_at | TIMESTAMPTZ DEFAULT NOW() | |

Unique constraint: `UNIQUE(user_id, slug)`

Postgres ENUM type (create in migration before the table):
```sql
CREATE TYPE instance_status AS ENUM ('disconnected', 'qr_pending', 'connected');
```

### 2. Session Manager (`src/sessions/manager.ts`)

This is the heart of the system. Each instance gets its own subdirectory under `SESSIONS_DIR` (e.g. `./sessions/usr_abc_main/`). Baileys' `useMultiFileAuthState` writes `creds.json` and a `keys/` folder there.

```typescript
import makeWASocket, {
  DisconnectReason,
  useMultiFileAuthState,
  fetchLatestBaileysVersion,
  makeCacheableSignalKeyStore,
  WASocket,
} from "@whiskeysockets/baileys";
import { Boom } from "@hapi/boom";
import pino from "pino";
import fs from "fs/promises";
import path from "path";
import { db } from "../db";
import { webhookDispatcher } from "../webhooks/dispatcher";
import { config } from "../config";

export type InstanceId = string;

export interface SessionMeta {
  socket: WASocket;
  userId: string;
  instanceId: InstanceId;
  sessionsDir: string;
  qr?: string;
  status: "qr_pending" | "connected" | "disconnected";
}

class SessionManager {
  private sessions = new Map<InstanceId, SessionMeta>();
  // Internal event subscribers for SSE
  private listeners = new Map<InstanceId, Set<(event: string, data: object) => void>>();

  private sessionDir(instanceId: InstanceId): string {
    return path.join(config.sessionsDir, instanceId);
  }

  async createSession(userId: string, instanceId: InstanceId): Promise<void> {
    if (this.sessions.has(instanceId)) return;

    const sessionsDir = this.sessionDir(instanceId);
    await fs.mkdir(sessionsDir, { recursive: true });

    const { state, saveCreds } = await useMultiFileAuthState(sessionsDir);
    const { version } = await fetchLatestBaileysVersion();

    const socket = makeWASocket({
      version,
      auth: {
        creds: state.creds,
        keys: makeCacheableSignalKeyStore(state.keys, pino({ level: "silent" })),
      },
      printQRInTerminal: false,
      logger: pino({ level: "silent" }),
      browser: ["WhatsApp API", "Chrome", "1.0.0"],
      syncFullHistory: false,
      generateHighQualityLinkPreview: true,
    });

    const meta: SessionMeta = {
      socket,
      userId,
      instanceId,
      sessionsDir,
      status: "disconnected",
    };

    this.sessions.set(instanceId, meta);

    socket.ev.on("creds.update", saveCreds);

    socket.ev.on("connection.update", async (update) => {
      const { connection, lastDisconnect, qr } = update;

      if (qr) {
        meta.qr = qr;
        meta.status = "qr_pending";
        await this.syncStatus(instanceId, "qr_pending");
        this.emit(instanceId, "qr", { qr });
        await webhookDispatcher.send(instanceId, "qr", { qr });
      }

      if (connection === "open") {
        meta.status = "connected";
        meta.qr = undefined;
        // Capture phone number from socket user JID
        const jid = socket.user?.id;
        const phone = jid ? jid.split(":")[0].split("@")[0] : null;
        await this.syncStatus(instanceId, "connected", phone ?? undefined);
        this.emit(instanceId, "connection.open", { phone });
        await webhookDispatcher.send(instanceId, "connection.open", { phone });
      }

      if (connection === "close") {
        const statusCode = (lastDisconnect?.error as Boom)?.output?.statusCode;
        const shouldReconnect = statusCode !== DisconnectReason.loggedOut;

        meta.status = "disconnected";
        this.sessions.delete(instanceId);
        await this.syncStatus(instanceId, "disconnected");
        this.emit(instanceId, "connection.close", { statusCode, shouldReconnect });
        await webhookDispatcher.send(instanceId, "connection.close", { statusCode, shouldReconnect });

        if (shouldReconnect) {
          setTimeout(() => this.createSession(userId, instanceId), 5_000);
        } else {
          // Logged out — purge session files from disk
          await fs.rm(sessionsDir, { recursive: true, force: true });
        }
      }
    });

    socket.ev.on("messages.upsert", async ({ messages, type }) => {
      if (type !== "notify") return;
      for (const msg of messages) {
        if (!msg.key.fromMe) {
          this.emit(instanceId, "message.received", { message: msg });
          await webhookDispatcher.send(instanceId, "message.received", { message: msg });
        }
      }
    });

    socket.ev.on("message-receipt.update", async (updates) => {
      this.emit(instanceId, "message.receipt", { updates });
      await webhookDispatcher.send(instanceId, "message.receipt", { updates });
    });
  }

  getSession(instanceId: InstanceId): SessionMeta | undefined {
    return this.sessions.get(instanceId);
  }

  async destroySession(instanceId: InstanceId, purge = false): Promise<void> {
    const meta = this.sessions.get(instanceId);
    if (!meta) return;
    try { await meta.socket.logout(); } catch {}
    this.sessions.delete(instanceId);
    if (purge) {
      await fs.rm(meta.sessionsDir, { recursive: true, force: true });
    }
  }

  // SSE subscription
  subscribe(instanceId: InstanceId, fn: (event: string, data: object) => void): () => void {
    if (!this.listeners.has(instanceId)) this.listeners.set(instanceId, new Set());
    this.listeners.get(instanceId)!.add(fn);
    return () => this.listeners.get(instanceId)?.delete(fn);
  }

  private emit(instanceId: InstanceId, event: string, data: object) {
    this.listeners.get(instanceId)?.forEach((fn) => fn(event, data));
  }

  private async syncStatus(instanceId: InstanceId, status: string, phoneNumber?: string) {
    const update: Record<string, unknown> = { status };
    if (phoneNumber) update.phone_number = phoneNumber;
    await db("instances").where({ id: instanceId }).update(update);
  }
}

export const sessionManager = new SessionManager();
```

### 3. Auth Middleware (`src/auth/middleware.ts`)

API keys are passed via `Authorization: Bearer <key>` header or `x-api-key` header.

```typescript
import { Request, Response, NextFunction } from "express";
import crypto from "crypto";
import { db } from "../db";

export interface AuthRequest extends Request {
  userId: string;
  keyId: string;
}

export async function requireApiKey(
  req: AuthRequest,
  res: Response,
  next: NextFunction
) {
  const raw =
    req.headers["x-api-key"] as string ||
    req.headers.authorization?.replace("Bearer ", "");

  if (!raw) {
    return res.status(401).json({ error: "API key required" });
  }

  const keyHash = crypto.createHash("sha256").update(raw).digest("hex");

  const apiKey = await db("api_keys")
    .where({ key_hash: keyHash, revoked: false })
    .first();

  if (!apiKey) {
    return res.status(401).json({ error: "Invalid or revoked API key" });
  }

  // Update last_used_at non-blockingly
  db("api_keys").where({ id: apiKey.id }).update({ last_used_at: new Date() }).catch(() => {});

  req.userId = apiKey.user_id;
  req.keyId = apiKey.id;
  next();
}

// Instance ownership guard — call after requireApiKey
export async function requireInstanceOwner(
  req: AuthRequest,
  res: Response,
  next: NextFunction
) {
  const instance = await db("instances")
    .where({ id: req.params.instanceId, user_id: req.userId })
    .first();

  if (!instance) {
    return res.status(404).json({ error: "Instance not found" });
  }

  (req as any).instance = instance;
  next();
}
```

### 4. API Key Generation (`src/auth/service.ts`)

Raw keys are shown **once** at creation time. Only the SHA-256 hash is stored.

```typescript
import crypto from "crypto";
import { db } from "../db";

export function generateApiKey(): string {
  // Format: wapi_<32 random bytes as hex>
  return `wapi_${crypto.randomBytes(32).toString("hex")}`;
}

export async function createApiKey(userId: string, label: string) {
  const raw = generateApiKey();
  const hash = crypto.createHash("sha256").update(raw).digest("hex");

  const [id] = await db("api_keys").insert({
    id: crypto.randomUUID(),
    user_id: userId,
    key_hash: hash,
    label,
    revoked: false,
    created_at: new Date(),
  });

  // Return the raw key — this is the only time it's shown
  return { id, key: raw, label };
}

export async function revokeApiKey(keyId: string, userId: string) {
  await db("api_keys")
    .where({ id: keyId, user_id: userId })
    .update({ revoked: true });
}
```

#### Webhook Payload Structure
Every webhook POST to your configured URL follows this envelope:
```json
{
  "instanceId": "usr_abc_main",
  "userId": "uuid",
  "event": "message.received",
  "timestamp": 1700000000,
  "data": { /* event-specific payload */ }
}
```

Supported event types:
- `qr` — new QR code available, `data.qr` is the raw QR string
- `connection.open` — instance connected to WA
- `connection.close` — disconnected, `data.statusCode` and `data.shouldReconnect`
- `message.received` — incoming message (full Baileys message object)
- `message.sent` — outgoing message confirmed
- `message.receipt` — delivery/read receipts
- `group.update` — group metadata changed
- `contact.update` — contact presence/name update

Webhook delivery includes a retry mechanism with exponential backoff (3 attempts).

## Webhook Dispatcher (`src/webhooks/dispatcher.ts`)

```typescript
import axios from "axios";
import { db } from "../db";

class WebhookDispatcher {
  async send(instanceId: string, event: string, data: object) {
    const instance = await db("instances").where({ id: instanceId }).first();
    if (!instance?.webhook_url) return;

    const subscribedEvents: string[] = instance.webhook_events || [];
    if (subscribedEvents.length > 0 && !subscribedEvents.includes(event)) return;

    const payload = {
      instanceId,
      userId: instance.user_id,
      event,
      timestamp: Math.floor(Date.now() / 1000),
      data,
    };

    const maxAttempts = 3;
    for (let attempt = 1; attempt <= maxAttempts; attempt++) {
      try {
        await axios.post(instance.webhook_url, payload, {
          timeout: 10_000,
          headers: { "Content-Type": "application/json" },
        });
        return;
      } catch {
        if (attempt < maxAttempts) {
          await new Promise((r) => setTimeout(r, attempt * 2000));
        }
      }
    }
  }
}

export const webhookDispatcher = new WebhookDispatcher();
```

## Session Restoration on Startup

On server start, query Postgres for all non-disconnected instances and re-initialise their Baileys sockets. The session files are already on disk from the previous run — `useMultiFileAuthState` picks them up automatically.

```typescript
// src/index.ts
import path from "path";
import fs from "fs/promises";
import { config } from "./config";
import { db } from "./db";
import { sessionManager } from "./sessions/manager";

async function restoreActiveSessions() {
  const instances = await db("instances")
    .whereIn("status", ["connected", "qr_pending"])
    .select("id", "user_id", "sessions_dir");

  for (const inst of instances) {
    // Only restore if the session directory actually exists on disk
    const dirExists = await fs.access(inst.sessions_dir)
      .then(() => true)
      .catch(() => false);

    if (!dirExists) {
      // Session files missing — reset status in DB
      await db("instances").where({ id: inst.id }).update({ status: "disconnected" });
      continue;
    }

    // Non-blocking: don't await, let sockets initialise in parallel
    sessionManager.createSession(inst.user_id, inst.id).catch((err) => {
      console.error(`Failed to restore session ${inst.id}:`, err.message);
    });
  }
}

app.listen(config.port, async () => {
  await db.migrate.latest();     // run pending migrations on startup
  await restoreActiveSessions();
  console.log(`API running on :${config.port}`);
});
```

## Rate Limiting & Security Recommendations

1. **Rate limit per API key** — use `express-rate-limit` with an in-process memory store (or `rate-limit-postgresql` for multi-process deployments), keyed on the `x-api-key` header value
2. **Instance caps per user** — enforce a max of N instances (e.g. 5) via a DB count check inside `POST /instances` before inserting
3. **Webhook signature** — sign every webhook payload with HMAC-SHA256 using the per-instance `webhook_secret` column:
   ```
   X-Webhook-Signature: sha256=<hmac>
   ```
4. **Input validation** — use `zod` for all request bodies; validate JID format before any `sendMessage` call
5. **Phone number normalisation** — strip `+`, spaces, dashes; append `@s.whatsapp.net` for DMs, `@g.us` for groups
6. **Session directory permissions** — restrict the `sessions/` directory to `700` (owner read/write only); each instance subdir at `750`
7. **Never log message content** — log only message IDs, JIDs, and timestamps; use `pino` redact paths for sensitive fields
8. **Postgres connection security** — use `sslmode=require` in `DATABASE_URL` for any non-localhost Postgres connection

## QR Code Delivery Options

Support two modes for QR delivery to clients:

1. **Polling** — `GET /instances/:id/qr` returns base64 PNG; client polls every 5s
2. **Webhook push** — `qr` event fired on webhook URL as soon as QR is generated
3. **SSE stream** — `GET /instances/:id/events` (Server-Sent Events) pushes QR, status, and incoming messages in real time

For SSE:
```typescript
router.get("/:instanceId/events", requireApiKey, requireInstanceOwner, (req, res) => {
  res.setHeader("Content-Type", "text/event-stream");
  res.setHeader("Cache-Control", "no-cache");
  res.setHeader("Connection", "keep-alive");

  const send = (event: string, data: object) => {
    res.write(`event: ${event}\ndata: ${JSON.stringify(data)}\n\n`);
  };

  // Subscribe to session events for this instance
  const cleanup = sessionManager.subscribe(req.params.instanceId, send);
  req.on("close", cleanup);
});
```

## Key Implementation Notes

- **Baileys is not officially supported** by WhatsApp/Meta. Use for personal or internal business projects; check WhatsApp's ToS for your use case.
- **Multi-device protocol** — Baileys uses the WA multi-device protocol. Each `makeWASocket()` call is a full WA Web session.
- **Never share auth state between instances** — each `instanceId` must have its own `useMultiFileAuthState` directory. Sharing a directory causes credential conflicts.
- **Filesystem layout** — `sessions/{instanceId}/creds.json` holds the device identity; `sessions/{instanceId}/keys/` holds the Signal protocol pre-keys. Both must survive server restarts. Mount this directory on a persistent volume in containerised deployments.
- **Do not gitignore individual files** — add `sessions/` to `.gitignore` entirely, not per-file patterns. Committing `creds.json` exposes your WhatsApp session.
- **`syncFullHistory: false`** is critical for performance — Baileys will otherwise attempt to download all chat history on first connect, which is extremely slow and memory-intensive.
- **Postgres JSONB for `webhook_events`** — query subscribed events with `webhook_events @> '["message.received"]'::jsonb` if you need server-side filtering in future.
- **Message retries** — Baileys handles E2E Signal protocol retries internally; do not implement your own retry loop around `sendMessage`.
- **Group JIDs** end in `@g.us`; individual contacts end in `@s.whatsapp.net`. Always validate JID format before calling `sendMessage` — an invalid JID throws synchronously.
- **QR expiry** — QR codes expire in ~60 seconds. After expiry, Baileys emits a fresh `qr` event automatically; relay it immediately via SSE and webhook.
- **Knex migrations on startup** — calling `db.migrate.latest()` at boot is safe for single-process deployments. For clustered deployments, run migrations as a separate one-off process/job before starting the API.
